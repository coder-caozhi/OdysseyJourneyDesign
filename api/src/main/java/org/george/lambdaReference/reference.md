###  Java 函数式编程演进笔记

#### 1. 函数式接口
- **定义**：接口中**只包含一个抽象方法**的接口。
- **作用**：它是 Lambda 表达式和方法引用的基础类型（如 `Runnable`, `Comparator`, `Function` 等）。

#### 2. 匿名内部类
- **本质**：一次性使用的类实现。
    - 对接口而言：是一次性的实现。
    - 对父类而言：是一次性的继承。
- **机制**：它可以重写父类或接口的方法。
- **语法格式**：
    ```java
    new 父类或接口() {
        // 花括号内是方法体
        @Override
        public void method() {
            // 实现逻辑
        }
    };
    ```
- **特点**：每次使用都相当于对该接口或父类进行了一次具体的、临时的定义。

#### 3. Lambda 表达式
- **本质**：匿名内部类的**语法糖（简化版）**。
- **简化逻辑**：去掉了冗余的 `new`、类名、方法名等 boilerplate 代码，只保留了核心部分：
    - **参数列表**
    - **方法体**
- **对比**：
    - **匿名类**：`new Runnable() { public void run() { System.out.println("Hi"); } }`
    - **Lambda**：`() -> System.out.println("Hi")`

#### 4. 方法引用
- **本质**：Lambda 表达式的进一步简化。当 Lambda 的方法体仅仅是调用一个**已经存在的方法**时，就可以用方法引用来替代。
- **四种类型**：
    1. **对象的实例方法**
        - 格式：`实例对象::方法名`
        - 示例：`System.out::println`
    2. **类的静态方法**
        - 格式：`类名::方法名`
        - 示例：`Math::max`
    3. **类的实例方法（特殊）**
        - 格式：`类名::方法名`
        - **特点**：调用者不是确定的对象，而是作为参数传入的对象。
        - 示例：`String::length` (相当于 `s -> s.length()`)
    4. **构造方法引用**
        - 格式：`类名::new`
        - 示例：`HashMap::new`

#### 5. 核心思想与源码阅读
- **演进路线**：匿名内部类 → Lambda → 方法引用（代码越来越简洁，语义越来越清晰）。
- **配合技术**：
    - **泛型**：提供类型安全。
    - **PECS 原则**：
        - `? extends T`：用于生产者，只读。
        - `? super T`：用于消费者，只写。
    - **应用**：在阅读 JDK 源码（特别是集合框架和 Stream API）时，结合 PECS 原则理解泛型通配符和方法引用的配合非常重要。
